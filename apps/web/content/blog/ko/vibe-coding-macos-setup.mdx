## 🍎 macOS에서 바이브코딩 환경을 따로 세팅해야 하는 이유

바이브코딩 환경 세팅 관련 글을 검색하면 Windows 기준으로 작성된 가이드가 많습니다. 터미널 명령어, 패키지 관리자, 설치 경로가 macOS와 Windows에서 다르기 때문에 Windows용 가이드를 macOS에서 그대로 따라하면 에러가 발생합니다.

macOS에서 개발환경을 세팅할 때 가장 먼저 해야 할 일은 **Homebrew**라는 패키지 관리자를 설치하는 것입니다. Windows의 winget과 같은 역할인데, macOS에는 기본으로 포함돼 있지 않아서 직접 설치해야 합니다. Homebrew가 있으면 Git, Node.js, VS Code 같은 도구를 터미널 명령어 한 줄로 설치할 수 있습니다.

이 글은 macOS 사용자만을 위해 Homebrew 설치부터 첫 프로젝트 실행까지 전 과정을 안내합니다. macOS Ventura(13) 이상을 기준으로 작성했으며, Monterey(12)에서도 대부분 동일하게 동작합니다.

## 📌 시작 전 확인사항

환경 세팅을 시작하기 전에 두 가지를 먼저 확인하세요.

### 💻 macOS 버전 확인

화면 왼쪽 상단의 Apple 메뉴(🍎) → "이 Mac에 관하여"를 클릭하면 현재 macOS 버전을 확인할 수 있습니다. macOS Monterey(12) 이상이면 이 가이드의 모든 내용을 따라할 수 있습니다.

### 🔧 Xcode Command Line Tools 확인

macOS에서 개발 도구를 설치하려면 Xcode Command Line Tools가 필요합니다. 이것은 Xcode 전체를 설치하는 것이 아니라, 컴파일러와 기본 개발 도구만 설치하는 가벼운 패키지입니다. 터미널을 열고 아래 명령어를 실행하세요.

```bash
xcode-select --version
```

버전 번호가 나오면 이미 설치돼 있는 것입니다. "command not found"가 나오면 아래 명령어로 설치합니다.

```bash
xcode-select --install
```

팝업 창이 뜨면 "설치"를 클릭하세요. 설치에 몇 분 정도 걸립니다. 완료된 후 다시 `xcode-select --version`을 실행해서 버전 번호가 나오면 정상입니다.

## 🍺 1단계: Homebrew 설치

Homebrew는 macOS의 패키지 관리자입니다. App Store에서 앱을 설치하듯이, 터미널에서 명령어 한 줄로 개발 도구를 설치하고 관리할 수 있게 해줍니다.

### 📥 설치 방법

터미널을 열고 아래 명령어를 붙여넣어 실행하세요.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

설치 중 비밀번호를 물어봅니다. macOS 로그인 비밀번호를 입력하세요. 입력할 때 화면에 아무것도 표시되지 않지만 정상입니다. 입력 후 Enter를 누르면 됩니다.

### 📌 Apple Silicon(M1/M2/M3/M4) 사용자 추가 설정

Apple Silicon Mac을 사용하는 경우, 설치가 끝난 뒤 터미널에 표시되는 안내 메시지를 반드시 확인하세요. 아래와 비슷한 두 줄의 명령어를 실행하라는 안내가 나옵니다.

```bash
echo >> ~/.zprofile
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

이 명령어를 실행하지 않으면 터미널을 새로 열 때마다 `brew` 명령어를 찾을 수 없다는 에러가 발생합니다. Intel Mac에서는 이 과정이 필요 없습니다.

### ✅ 설치 확인

터미널을 닫고 새로 연 뒤 아래 명령어를 실행하세요.

```bash
brew --version
```

`Homebrew 4.x.x` 형태의 버전 번호가 나오면 정상입니다.

## 🔧 2단계: Git 설치

Git은 코드의 변경 이력을 관리하는 도구입니다. 바이브코딩에서 AI에게 여러 번 수정을 요청할 때, 원하는 시점으로 되돌릴 수 있게 해주는 안전장치입니다.

macOS에는 Xcode Command Line Tools를 통해 Git이 기본 포함돼 있을 수 있습니다. 하지만 버전이 오래된 경우가 많으므로 Homebrew로 최신 버전을 설치하는 것을 권장합니다.

### 📥 Homebrew로 설치

```bash
brew install git
```

설치가 끝나면 터미널을 닫고 새로 열어주세요.

### ✅ 설치 확인

```bash
git --version
```

`git version 2.x.x`가 나오면 성공입니다. 만약 Homebrew로 설치한 최신 버전이 아니라 기존 시스템 Git이 표시된다면, `which git`을 실행해서 경로가 `/opt/homebrew/bin/git`(Apple Silicon) 또는 `/usr/local/bin/git`(Intel)인지 확인하세요. `/usr/bin/git`이 나오면 시스템 기본 Git을 사용 중인 것이므로, 터미널을 재시작하면 Homebrew 버전으로 전환됩니다.

### 📌 Git 초기 설정

설치 후 사용자 이름과 이메일을 등록합니다. 코드 변경사항을 저장(커밋)할 때 누가 변경했는지 기록하는 용도입니다.

```bash
git config --global user.name "내 이름"
git config --global user.email "my@email.com"
```

## 📗 3단계: Node.js 설치

Node.js는 JavaScript 코드를 실행하는 환경입니다. AI가 만들어준 웹 프로젝트를 내 컴퓨터에서 돌려보려면 반드시 필요합니다.

### 📥 Homebrew로 설치 (추천)

```bash
brew install node@22
```

이 명령어는 Node.js 22 LTS(안정 버전)를 설치합니다. `node@22`처럼 버전을 지정하면 안정적인 LTS 버전을 사용할 수 있습니다.

설치 후 아래 명령어로 PATH를 설정해야 할 수 있습니다. 터미널에 표시되는 안내 메시지를 확인하세요.

```bash
echo 'export PATH="/opt/homebrew/opt/node@22/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 📥 nvm으로 설치 (여러 버전 관리가 필요한 경우)

프로젝트마다 다른 Node.js 버전이 필요한 경우 nvm(Node Version Manager)을 사용합니다. 바이브코딩 입문 단계에서는 Homebrew 설치로 충분하지만, 나중에 여러 프로젝트를 다룰 때 유용합니다.

```bash
brew install nvm
mkdir ~/.nvm
```

설치 후 `~/.zshrc` 파일에 아래 내용을 추가합니다.

```bash
echo 'export NVM_DIR="$HOME/.nvm"' >> ~/.zshrc
echo '[ -s "/opt/homebrew/opt/nvm/nvm.sh" ] && \. "/opt/homebrew/opt/nvm/nvm.sh"' >> ~/.zshrc
source ~/.zshrc
```

그 다음 Node.js LTS 버전을 설치합니다.

```bash
nvm install --lts
nvm use --lts
```

### ✅ 설치 확인

```bash
node -v
npm -v
npx -v
```

세 가지 모두 버전 번호가 나오면 정상입니다. npm과 npx는 Node.js를 설치하면 자동으로 함께 설치됩니다.

## 💙 4단계: VS Code 설치

VS Code(Visual Studio Code)는 코드를 작성하고 편집하는 프로그램입니다. AI 코딩 확장 프로그램을 설치할 수 있는 가장 범용적인 편집기입니다.

### 📥 Homebrew로 설치 (추천)

```bash
brew install --cask visual-studio-code
```

`--cask`는 GUI 애플리케이션을 설치할 때 사용하는 옵션입니다. 설치가 끝나면 응용 프로그램 폴더에서 VS Code를 찾을 수 있습니다.

### 📥 공식 사이트에서 설치

[VS Code 공식 사이트](https://code.visualstudio.com)에서 macOS용 설치 파일(.dmg)을 다운로드합니다. 다운로드한 파일을 열고 VS Code 아이콘을 응용 프로그램 폴더로 드래그하면 설치 완료입니다.

### 📌 터미널에서 `code` 명령어 활성화

VS Code를 설치한 뒤 터미널에서 `code .` 명령어로 현재 폴더를 바로 열 수 있도록 설정합니다. VS Code를 실행한 뒤 `Cmd+Shift+P`를 눌러 명령 팔레트를 열고, "Shell Command: Install 'code' command in PATH"를 선택하세요.

### ✅ 설치 확인

터미널을 닫고 새로 연 뒤 아래 명령어를 실행하세요.

```bash
code --version
```

버전 번호가 나오면 정상입니다.

## ✅ 전체 점검

네 가지 도구가 모두 설치됐는지 한 번에 확인합니다. 터미널을 새로 열고 아래 명령어를 순서대로 실행하세요.

```bash
brew --version
git --version
node -v
npm -v
code --version
```

| 명령어 | 정상 결과 | 실패 시 원인 |
|--------|----------|-------------|
| `brew --version` | `Homebrew 4.x.x` | Homebrew 미설치 또는 PATH 미설정 |
| `git --version` | `git version 2.x.x` | Git 미설치 |
| `node -v` | `v22.x.x` 등 | Node.js 미설치 또는 PATH 미설정 |
| `npm -v` | `10.x.x` 등 | Node.js 설치 불완전 |
| `code --version` | 버전 번호 | VS Code PATH 미등록 |

다섯 가지 모두 버전 번호가 나오면 개발환경 세팅이 완료된 것입니다.

## 🚀 첫 프로젝트로 최종 검증

도구가 설치됐더라도 실제 프로젝트를 만들어봐야 확실하게 검증할 수 있습니다.

### 📁 프로젝트 생성

원하는 위치에서 터미널을 열고 아래 명령어를 실행하세요.

```bash
npx create-next-app@latest my-first-site
```

여러 옵션을 물어보는데, 모두 기본값(Enter)으로 진행하면 됩니다.

### ▶️ 개발 서버 실행

```bash
cd my-first-site
npm run dev
```

터미널에 `http://localhost:3000`이 표시됩니다. 이 주소를 웹 브라우저에 붙여넣어 Next.js 기본 페이지가 나타나면 모든 환경 세팅이 정상입니다. 개발 서버를 종료하려면 `Ctrl+C`를 누르세요.

### 🎯 VS Code에서 프로젝트 열기

프로젝트 폴더에서 아래 명령어를 실행하면 VS Code가 열립니다.

```bash
code .
```

VS Code가 열리면 `` Ctrl+` ``(백틱)을 눌러 내장 터미널을 열 수 있습니다. 앞으로는 별도 터미널을 열 필요 없이 VS Code 안에서 코드 편집과 명령어 실행을 모두 할 수 있습니다.

## ⚠️ macOS에서 자주 발생하는 문제와 해결

### 🚫 "확인되지 않은 개발자" 경고 (Gatekeeper)

macOS는 App Store 외부에서 다운로드한 앱을 처음 실행할 때 보안 경고를 표시합니다. VS Code를 처음 열 때 이 경고가 나타나면 "열기"를 클릭하세요. 그래도 열리지 않으면 시스템 설정 → 개인 정보 보호 및 보안 → 하단의 "확인 없이 열기"를 클릭합니다.

### 🚫 brew 명령어를 찾을 수 없음

Apple Silicon Mac에서 Homebrew 설치 후 이 문제가 발생하면 PATH 설정이 안 된 것입니다. 아래 명령어를 실행하세요.

```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
```

Intel Mac에서는 Homebrew가 `/usr/local/bin`에 설치되므로 보통 PATH 문제가 발생하지 않습니다.

### 🚫 zsh: permission denied

스크립트 실행 시 권한 문제가 발생하면 파일에 실행 권한이 없는 것입니다. 아래 명령어로 실행 권한을 부여할 수 있습니다.

```bash
chmod +x 파일명.sh
```

### 🚫 node 명령어가 인식되지 않음

Homebrew로 `node@22`를 설치한 경우 PATH에 자동 추가되지 않을 수 있습니다. 터미널에 표시된 안내 메시지에 따라 PATH를 수동으로 추가하세요. 또는 `brew install node`로 최신 버전을 설치하면 PATH가 자동으로 설정됩니다.

### 🚫 Xcode 라이선스 동의 필요

`git` 등의 명령어 실행 시 "Agreeing to the Xcode/iOS license requires admin privileges"라는 메시지가 나오면 아래 명령어를 실행하세요.

```bash
sudo xcodebuild -license accept
```

## 💡 macOS 개발환경 팁

### 📌 터미널은 zsh가 기본

macOS Catalina(10.15)부터 기본 셸이 bash에서 **zsh**로 변경됐습니다. 설정 파일도 `~/.bashrc` 대신 `~/.zshrc`를 사용합니다. 인터넷에서 bash용으로 작성된 가이드를 따라할 때 `.bashrc` 대신 `.zshrc`에 설정을 추가해야 합니다.

### 📌 Homebrew로 설치한 도구 업데이트

Homebrew로 설치한 도구들은 아래 명령어로 한 번에 업데이트할 수 있습니다.

```bash
brew update && brew upgrade
```

주기적으로 실행하면 보안 패치와 새 기능을 최신 상태로 유지할 수 있습니다.

### 📌 폴더 경로에 한글이나 공백 주의

프로젝트를 만들 폴더 경로에 한글이나 공백이 있으면 일부 도구에서 에러가 발생할 수 있습니다. `~/Desktop/내 프로젝트` 대신 `~/projects/my-first-site`처럼 영문 경로를 사용하는 것이 안전합니다.

### 📌 Spotlight 검색으로 앱 빠르게 열기

`Cmd+Space`를 누르면 Spotlight 검색이 열립니다. "Terminal"이나 "Visual Studio Code"를 입력하면 앱을 빠르게 찾아 실행할 수 있습니다.
