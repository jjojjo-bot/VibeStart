## 🤔 왜 이 비교가 필요한가

바이브코딩을 시작하려면 AI 코딩 도구가 필요합니다. 가장 많이 언급되는 도구가 Cursor와 Claude Code인데, 둘은 작동 방식이 완전히 다릅니다. "어떤 걸 먼저 써야 하지?"라는 질문에 답하기 위해 비전공자 관점에서 두 도구를 비교합니다.

결론부터 말하면, 둘 다 쓰는 것이 가장 좋습니다. 하지만 하나만 먼저 시작해야 한다면, 자신의 작업 스타일에 맞는 도구를 고르는 것이 중요합니다.

## 📋 한눈에 비교

| 항목 | Cursor | Claude Code |
|------|--------|-------------|
| 형태 | GUI 코드 에디터 (VS Code 기반) | CLI 터미널 도구 |
| 설치 방식 | 앱 다운로드 설치 | `npm install -g` 명령어 |
| AI 사용 방식 | 에디터 안에서 채팅/인라인 편집 | 터미널에서 대화 |
| 코드 수정 | AI가 제안 → 사용자가 수락/거부 | AI가 직접 파일 수정 (권한 확인 후) |
| 프로젝트 생성 | AI가 명령어 제안 → 사용자가 실행 | AI가 직접 명령어 실행 |
| 코드 확인 | diff로 변경 사항 시각적 확인 | 터미널에서 변경 로그 확인 |
| 무료 사용 | 월 2,000 자동완성 + 50 프리미엄 요청 | API 사용량만큼 과금 (최소 비용 가능) |
| 유료 플랜 | Pro $20/월 | API 종량제 또는 Claude Max $100~$200/월 |
| Node.js 필요 | 프로젝트 실행 시 필요 | 설치 자체에 필요 |
| 추천 대상 | 코드를 직접 보면서 배우고 싶은 사람 | AI에게 최대한 맡기고 싶은 사람 |

## 🖥️ 작업 방식의 차이

### Cursor: 에디터 중심

Cursor는 VS Code와 동일한 화면에서 작업합니다. 왼쪽에 파일 탐색기, 가운데에 코드 편집 영역, 오른쪽에 AI 채팅 패널이 배치됩니다. 코드를 직접 보면서 AI에게 특정 부분의 수정을 요청하는 방식입니다.

작업 흐름 예시:

1. 파일을 열고 수정할 부분을 확인합니다.
2. `Ctrl+L`로 채팅을 열고 "이 컴포넌트에 로딩 상태를 추가해줘"라고 요청합니다.
3. AI가 코드 변경 사항을 diff로 보여줍니다.
4. 변경 내용을 확인하고 수락 또는 거부합니다.

코드가 눈에 보이기 때문에 "AI가 무엇을 바꿨는지" 바로 확인할 수 있습니다. 코드를 이해하면서 배우고 싶은 사람에게 적합합니다.

### Claude Code: 터미널 중심

Claude Code는 터미널 창에서 AI와 대화합니다. 별도 에디터 화면 없이 텍스트로 요청하고, AI가 파일을 직접 생성하고 수정합니다.

작업 흐름 예시:

1. 터미널에서 `claude`를 실행합니다.
2. "로그인 페이지를 만들어줘. 이메일과 비밀번호 입력 필드가 있고, 반응형으로 해줘"라고 입력합니다.
3. Claude가 실행 계획을 보여주고, 승인하면 파일을 직접 생성/수정합니다.
4. 브라우저에서 결과를 확인합니다.

AI가 프로젝트 전체를 관리하므로 한 번에 여러 파일을 수정할 수 있습니다. "내가 원하는 것을 말하면 AI가 만들어주는" 경험에 더 가깝습니다.

## 💰 비용 비교

### Cursor

| 플랜 | 월 비용 | 포함 내용 |
|------|---------|----------|
| Hobby (무료) | $0 | 자동완성 2,000회, 느린 프리미엄 요청 50회 |
| Pro | $20 | 무제한 자동완성, 빠른 프리미엄 요청 500회 |

무료 플랜으로 시작해서 부족하면 Pro로 업그레이드하면 됩니다. 비용이 예측 가능하다는 것이 장점입니다.

### Claude Code

| 방법 | 월 비용 | 특징 |
|------|---------|------|
| API 종량제 | 사용량에 따라 다름 (입문 단계 $1~$5) | 쓴 만큼만 과금, 한도 설정 가능 |
| Claude Max | $100~$200 | 구독형, 사용량 한도 내 무제한 |

입문 단계에서 API 종량제를 사용하면 월 $5 이내로 시작할 수 있습니다. 다만 사용량이 늘면 비용도 증가하므로 Anthropic 콘솔에서 한도를 설정해두는 것이 좋습니다.

### 비용 요약

가볍게 시작한다면 둘 다 낮은 비용으로 사용할 수 있습니다. Cursor는 무료 플랜이 있어 초기 비용이 0이고, Claude Code는 API 종량제로 $1~$5 정도면 입문에 충분합니다.

## ✅ 각 도구의 장점

### Cursor의 장점

- **시각적 피드백**: 코드 변경 사항을 diff로 바로 확인할 수 있습니다.
- **낮은 진입 장벽**: 앱을 설치하면 바로 시작할 수 있습니다. 터미널 지식이 없어도 됩니다.
- **VS Code 호환**: VS Code의 확장 프로그램, 설정, 테마를 그대로 사용할 수 있습니다.
- **무료 플랜**: 비용 부담 없이 시작할 수 있습니다.
- **인라인 편집**: 코드의 특정 부분만 선택해서 수정 요청이 가능합니다.

### Claude Code의 장점

- **높은 자율성**: AI가 프로젝트 구조 설계부터 파일 생성, 코드 작성까지 주도적으로 처리합니다.
- **대규모 작업에 강함**: 여러 파일을 한 번에 생성하고 수정할 수 있습니다.
- **자연어 중심**: 코드를 몰라도 원하는 것을 말로 설명하면 됩니다.
- **터미널 명령 실행**: 패키지 설치, 서버 실행 등 터미널 작업도 AI가 대신합니다.
- **CLAUDE.md 활용**: 프로젝트 규칙을 파일로 정의해두면 일관된 코드를 생성합니다.

## ⚠️ 각 도구의 단점

### Cursor의 단점

- **수동 작업 필요**: AI가 제안한 코드를 하나씩 확인하고 수락해야 합니다. 대규모 변경 시 번거로울 수 있습니다.
- **프로젝트 구조 설계 한계**: AI가 전체 프로젝트를 한 번에 만들어주지는 않습니다.
- **무료 사용량 제한**: 월 50회 프리미엄 요청은 적극적으로 사용하면 빠르게 소진됩니다.

### Claude Code의 단점

- **터미널 필수**: 터미널에 익숙하지 않으면 처음에 어려울 수 있습니다.
- **설치 과정 복잡**: Node.js 설치, API 키 발급, 환경 변수 설정 등 초기 설정이 필요합니다.
- **비용 예측 어려움**: API 종량제는 사용량에 따라 비용이 달라져서 예상하기 어렵습니다.
- **코드 확인 불편**: 터미널에서 변경 내용을 확인하는 것이 에디터의 diff보다 직관적이지 않습니다.

## 🎯 상황별 추천

### Cursor를 먼저 사용하면 좋은 경우

- 터미널을 아직 한 번도 써본 적 없는 경우
- 코드가 어떻게 생겼는지 보면서 배우고 싶은 경우
- 비용을 0원으로 시작하고 싶은 경우
- VS Code를 이미 사용 중이거나 사용법을 알고 있는 경우

### Claude Code를 먼저 사용하면 좋은 경우

- 터미널 기본 사용법을 이미 알고 있는 경우
- 코드를 직접 쓰기보다 AI에게 최대한 맡기고 싶은 경우
- 프로젝트 전체를 빠르게 만들고 싶은 경우
- 여러 파일에 걸친 큰 변경 작업이 많은 경우

### 둘 다 함께 사용하는 경우 (추천)

실제로 가장 효율적인 방법은 두 도구를 함께 사용하는 것입니다.

| 작업 | 추천 도구 | 이유 |
|------|----------|------|
| 새 프로젝트 생성 | Claude Code | 프로젝트 구조를 한 번에 만들 수 있음 |
| 대규모 기능 추가 | Claude Code | 여러 파일을 동시에 생성/수정 |
| 특정 파일 세부 수정 | Cursor | diff로 변경 사항을 시각적으로 확인 |
| 디자인/스타일 조정 | Cursor | 코드를 보면서 미세 조정 |
| 에러 디버깅 | 둘 다 가능 | 에러 메시지를 붙여넣으면 해결 방법 제안 |
| 코드 이해/학습 | Cursor | 코드를 보면서 설명을 요청 |

Claude Code로 프로젝트 뼈대를 만들고, Cursor에서 세부 사항을 다듬는 워크플로가 가장 자연스럽습니다.

## 🔄 전환하기 쉬운가?

두 도구는 같은 프로젝트 폴더를 공유할 수 있습니다. Claude Code로 프로젝트를 만든 뒤 Cursor에서 같은 폴더를 열면 됩니다. 반대도 마찬가지입니다. 파일은 로컬에 저장되므로 어느 도구에서 수정해도 다른 도구에서 바로 반영됩니다.

전환 시 특별한 설정이나 변환 과정은 필요 없습니다. 프로젝트 폴더만 같으면 자유롭게 오갈 수 있습니다.

## 💡 입문자를 위한 시작 전략

1. **1단계**: Cursor를 설치하고 무료 플랜으로 시작합니다. AI 채팅으로 간단한 페이지를 만들어봅니다.
2. **2단계**: 터미널 사용이 익숙해지면 Claude Code를 설치합니다. API 키를 발급받고 한도를 $5로 설정합니다.
3. **3단계**: 새 프로젝트는 Claude Code로 생성하고, 세부 수정은 Cursor에서 합니다.
4. **4단계**: 자신에게 맞는 비율을 찾아갑니다. 어떤 사람은 Claude Code 위주로, 어떤 사람은 Cursor 위주로 작업합니다.

두 도구 모두 처음 사용법을 익히는 데 30분이면 충분합니다. 부담 갖지 말고 둘 다 시도해보세요.
